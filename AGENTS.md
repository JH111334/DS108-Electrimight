# AGENTS.md - DS108-Electrimight

## Scope

These instructions apply to the whole repository. Follow them before making code,
data, notebook, Markdown, LaTeX, or report changes.

## Project Context

- Project: DS108-Electrimight, a steel-industry electricity consumption
  preprocessing and dataset-building project.
- Main goals: data quality audit, weather integration, time/frequency/physical
  feature engineering, anomaly labeling, GAN augmentation, reproducibility, and
  IEEE-style reporting.
- Language: Vietnamese documentation and reports must use full Vietnamese
  diacritics.
- Runtime target: Python 3.12 on Windows PowerShell.
- Data dates: parse CSV dates with `dayfirst=True` for `DD/MM/YYYY` input.

## Mandatory Encoding Rules

- Save all text files as UTF-8 without BOM: `.md`, `.py`, `.txt`, `.csv`, `.tex`,
  `.bib`, `.json`, `.yml`, and extracted notebook text.
- Do not use `cp1252`, `latin-1`, `iso-8859-1`, or other legacy Windows
  single-byte encodings.
- In Python file I/O, always pass `encoding="utf-8"` explicitly:

  ```python
  with open("file.md", "w", encoding="utf-8") as f:
      f.write(content)
  ```

- If a file contains mojibake such as `Gi��i thi���u`, rewrite the affected
  content with correct Vietnamese diacritics in UTF-8. Do not preserve or
  manually patch broken bytes in place.

## Windows UTF-8 Workflow

- Avoid printing Vietnamese text directly to PowerShell stdout. Redirect output
  to files or inspect content through UTF-8-safe tooling.
- When Shell work is unavoidable, initialize UTF-8 first:

  ```powershell
  [Console]::OutputEncoding = [System.Text.Encoding]::UTF8
  $env:PYTHONIOENCODING = "utf-8"
  ```

- Prefer repository tools such as `tools/utf8_replace.py` for text replacement.
- Temporary scripts or outputs must be removed before finishing:
  `_temp_*`, `_*.py`, `*_extracted.txt`, and scratch output files.

## Data Handling Contract

- Never overwrite raw source data in `data/bronze/` as part of routine pipeline
  work.
- Preserve row alignment and timestamp semantics when merging features.
- Weather data is exogenous. Resample hourly weather to 15-minute intervals
  before merging with steel consumption records.
- Keep label columns auditable: anomaly flags, confidence scores, and explanation
  fields must remain interpretable.
- Do not introduce target leakage. Any feature using future information or label
  leakage must be removed or clearly isolated from modeling.
- When changing schema, update the relevant metadata:
  `metadata/dataset/CODEBOOK.csv`,
  `metadata/dataset/DS108_Gold_Feature_Catalog.md`,
  `metadata/dataset/DATASHEET.md`, and labeling guidance if label logic changed.

## Python Standards

- Use `pathlib.Path` for filesystem paths where practical.
- Keep data transformations deterministic and reproducible.
- Prefer explicit column lists and validation assertions over implicit dataframe
  mutations.
- Keep functions small enough to test. Add docstrings for public functions and
  non-obvious domain calculations.
- Use existing project modules and naming conventions before adding new
  abstractions.
- Do not silently swallow data errors. Fail with actionable messages when required
  columns, invalid date parsing, impossible physical values, or row-count changes
  appear.

## IEEE and LaTeX Report Rules

- Primary report guidance lives in `references/documents/` and
  `references/report-guides/`.
- For Vietnamese LaTeX, prefer XeLaTeX or LuaLaTeX with `fontspec` when possible:

  ```latex
  \usepackage{fontspec}
  \usepackage[vietnamese]{babel}
  ```

- If using pdfLaTeX, include UTF-8 and Vietnamese font encoding:

  ```latex
  \usepackage[utf8]{inputenc}
  \usepackage[T5]{fontenc}
  \usepackage[vietnamese]{babel}
  ```

- IEEE style expectations:
  - Abstract: 150-250 words.
  - Keywords: 3-5 terms.
  - Citations: numbered IEEE format such as `[1]`, `[2]`, `[3]--[5]`.
  - Figures: caption below with `Fig. 1.`.
  - Tables: caption above with `TABLE I`.
  - Escape underscores and bare ampersands in text/table content.
